BattleBoats

by: Shivam Pansuria

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I was curious about the process and work into making a game. My decision to make this game as opposed to any other game is that I've always wanted to make a game where you avoid hitting bombs and you're playing against another player.

2. What I Learned

What I learned from the user testing was that I've played the game so much, that I didn't know how difficult the game was.

3. Architecture of My Game

The game works with a bomb dropping from the top. Both ships must shoot projectiles to destroy the bombs with a point being given every time a bomb is destroyed. A present falls 10 percent of the time with it giving a voice command and a point. The classes interact with the speak class being used when the ships intersect with it. The powerup class is used when the powerup is created and drawn.

4. Technical Implementation

There's a 50 percent chance of bombs dropping from the top of the screen. There's a 10 percent chance of a present dropping. When deciding what powerup to give to the player, the code runs through a random to determine if it's a good or bad powerup. On the end screen, there's an option to play an inspirational quote to cheer you up which is also coded by using a next int.

5. Most Challenging Part

The most challenging part of this project was having to pick from my ideas and decide which ones I can really implement into the game without it taking an unrealistic amount of time. This also included cutting down on parts of my vision in the game. I overcame these challenges by finding new ways to add creative things into the game without making it too complicated.

6. Future Fixes and Updates

I would like to add more variants of presents and fix small issues such as the ship going a bit past the divider before stopping. If I was to continue to work on this game, I would have more people test it out and get more feedback.
